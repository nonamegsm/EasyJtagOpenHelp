eMMC Page

The eMMC Page - work directly with the eMMC chip connected to the interface.

The upper part of the Connect and Identify eMMC window contains the Detect button - check the connection to eMMC and the settings of the connection interface:

Box Pinout Mode - select the type of connection eMMC (ISP / E-Socket);
eMMC CLK Rate - frequency of work with eMMC;
eMMC Voltage - select the voltage IO eMMC, it must correspond to the hardware power supply IO eMMC (fixed for E-Socket 2.8v);
eMMC Bus Width - selection of the operating mode with eMMC by the number of data lines (1/4/8);

Show Less Debug Data - allows you to reduce the log without displaying unused information eMMC in the log window;
Read soft Partition - determine and log the availability and information about eMMC partitions by clicking on the Detect button;
HiPower (TM) CLK Drive mode - activation of the HiPower (TM) CLK mode for connecting to a working device (for details, see "connection information"), enable only when using this mode.

The lower part of the window contains 5 tabs of working with eMMC:

1) Flasher - the main window for reading / deleting / writing data from / to eMMC;

ROM to access: - selection of the hardware of the eMMC area;
Preset - preset sizes for Start / Lenght;
Start - the starting address of the area;
Lenght - is the length of the area;

Write All Roms - an option for recording several files with "one button", active when selecting "Write all eMMC Regions at once";
Read eMMC Card - reading from eMMC into a file of the selected area;
Write eMMC Card - record in eMMC the selected area from the file selected in "Image";
Erase eMMC Card - erases the selected area of eMMC;
Stop Operation - button for interrupting / canceling the performed read / erase / write operations;

Image - select a file for recording "Write eMMC Card";
ROM1 / ROM2 / ROM3 / extCSD - select files to write to the appropriate sections in the "Write All Roms" mode;

Limit write lengh - limit the write limit in eMMC to its size;
Verify write / erase operation - enable verification of the operation to erase the recording (slows down the operation by half);
Upload Pinout on server - button for sending to the server the ISP pinout  from user.

 

2) eMMC BOOT Cfg - configuring eMMC boot modes through editing extCSD registers.

Boot_Partition_EN - selection of the boot partition from the eMMC (setting the bit register 179 extCSD);
Boot_ACK - setting the automatic boot sending mode to the CPU (setting the bit register 179 extCSD);
Set Partition Config: - record the selected value of register 179 to extCSD;

Boot_Mode - selection of the Data bus mode when working with Boot eMMC (setting the bit register 177 extCSD);
Boot_Bus_Width - selection of the width of the Data bus when working with Boot eMMC (setting the bit register 177 extCSD);
Reset_Boot_Bus_Width - selects the Data bus reset mode when working with Boot eMMC (setting the bit register 177 extCSD);
Set Boot Width Config: - record the selected value of register 177 in extCSD;

Ext_CSD_RST_N_FUNC - selection of the operating mode of the physical signal RST_N eMMC (setting the bit register 162 extCSD);
Set RST Conf - record the selected value of register 162 in extCSD;

Set i9300 / N7100 boot config - write registers 179/177 (with values ​​0x48 / 0x01, respectively) for Exynos processors (Samsung).

 

3) Advanced Options - work with eMMC, not included in other types of work (by tabs).

Android FRP clear - removal of FRP in the firmware recorded in eMMC;
Clear PIN / GTS / PASS - removal of the request for password and graphic key in Android firmware recorded in eMMC;
Android FDI Info - information about the presence of encrypted partitions and FRP in the firmware recorded in eMMC;

Dump eMMC FW - reading Samsung eMMC internal firmware for sending to the server;
Update eMMC FW - updating Samsung eMMC internal firmware or restoring Samsung eMMC internal firmware ( by Testpoint).

Custom Operation:

- Universal: Repair extCSD - recording extCSD backups in eMMC;

- Universal: Factory Format - erase eMMC quickly (sometimes it does not work on some eMMC);

- SAMSUNG: Read S.M.A.R.T. Report - reading information S.M.A.R.T. from Samsung eMMC;

- SAMSUNG: VTU Name '000000' Repair - restoration of Samsung eMMC KMVTU000LM-B503, which are defined without Testpoint with eMMC NAME: 00000;

- SAMSUNG: MoviNAND Format and Reset - reset with Samsung eMMC data format (Wipe);

- SAMSUNG: MoviNAND Factory Reset - reset with erasing Samsung eMMC data;

- SAMSUNG: Remove Write Protection - removal of Write Protection mode with Samsung eMMC;

- SAMSUNG: CID Editor - CID editor for Samsung eMMC;

- SANDISK: Read S.M.A.R.T. Report - reading information S.M.A.R.T. from eMMC Sandisk.

 

SD Tool - for eMMC or SDcard connected as eMMC:

Password - password field for eMMC / SDcard;

Set eMMC Password - setting a password for eMMC / SDcard;

Erase eMMC Password - erase the password and all data on eMMC / SDcard;

Search eMMC Password - bruteforce password eMMC / SDcard (in development);

Universal Format Repair - universal format recovery.

 

4) Hardware Partitions - creation and configuration of eMMC hardware partitions (ROM2 / 3, GP1-4).

Consists of two sections:

Samsung Boot+RPMB sizes - allows user to resize Boot1 / 2 (ROM2 / 3) and RPMB areas only for Samsung eMMC revisions below 5.1 (not including rev 5.1).

ROM2, ROM2 size, RPMB Block Size - selection of partition sizes;

Write to extCSD - write selected sizes to extCSD.

Important: when recording a new configuration, Wipe eMMC occurs, i.e. erasing all data from the eMMC, and on some eMMC the information on the GP1-4 partitions can be erased.

Important: if, after this operation, the sizes of ROM2 / 3 / RPMB do not correspond to the selected ones, this means that this microcircuit does not support the selected sizes and the operation should be repeated, reducing the sizes.

JEDEC GP Partition sizes - creation of partitions GP1-4 for chips in which the value of the extCSD register (155 / 0x9B) - PARTITON_SETTING_COMPLETED is equal to "0x00".


Important: the creation of partitions "GP1-4" is due to the reduction in the size of "User / ROM1" (cut off from it).

GP1-4 Part Size - selection of sizes of future sections;


ENH GP1-4 - increases the reliability of the partition for MLC / TLC memory by switching it to SLC / MLC mode. The amount of memory allocated for the partition is 2 times larger than the selected size;

WR_REL GP1-4 - sets the recording mode to the GP1-4 sections, which increases the reliability of data recording to prevent data loss during recording (the mode when the old data of the recorded block is not erased and new ones are written to the spare place);

User Enhanced Start / Size - turning the beginning of the User / ROM1 section into Enhanced mode, similar to ENH GP1-4 (address and length are selected);

Set WR_REL User - sets the recording mode in the same way as WR_REL GP1-4 for the User / ROM1 section.

Test Setting - write selected parameters to extCSD, without setting the extCSD register (155 / 0x9B) - PARTITON_SETTING_COMPLETED to "0x00";

Write to extCSD - the final record of the selected parameters in extCSD, with the extCSD register (155 / 0x9B) - PARTITON_SETTING_COMPLETED set to "0x01", after which operations to change the areas of "GP1-4" can not be changed.

 

5) OTP - One Time Programmable, contains Info  panel, Read and Write panel and OTP Config panel.

Info panel contain Scan for OTP button and OTP Offset box.

Scan for OTP  - scans entire eMMC address space for OTP (One Time Programmable) area or multiple OTP areas. Results are printed in a log window if such areas detected.

Read and Write panel contain OTP Data file box and  Read  OTP Region, Write OTP Region buttons:

OTP Data file - box selection for writing OTP area of eMMC;

Read OTP Region button - reads contents of OTP are in selected file;

Write OTP Region button - writes a file to OTP region by provided address. Operation is possible only if OTP region is not locked permanently.

OTP Config panel:

Lock OTP -  set protection flag for provided in Offset box address.

Unlock OTP - clear protection flag for provided on offset box address. Possible only if protection flag is not permanent. Otherwise can be cleared only by eMMC 4.5 firmware update of factory reset on Samsung eMMC chips.

Temporary: Write and Erase prevention scheme that can be enabled and disabled.

Power-on: Write and Erase prevention scheme, which once enabled, cannot be reversed untill a power failure event that causes the device to reboot occurs or the device is reset using the reset pin.

Permanent: Write and Erase prevention scheme, which once enabled, cannot be reversed.
