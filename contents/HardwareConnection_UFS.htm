Hardware Connection UFS

There are 2 types of UFS chips connection:

  • Socket connection (4 link pairs: 2x read + 2x write)
  • ISP (In-System Programing) connection (2 link pair: 1x read + 1x write)

 

UFS chips is work like ethernet - it's create link before host and chip and supports him constantly. UFS chips establishes a connection (link setting mode) always in PWM-G0 mode, after that the host (box) and the chip exchange information about the supported modes, and the operating mode switches to one of the supported.

 

Z3x EasyJTAG Plus box supports work with UFS chips in LS mode PWM-G4 (66MHz), which allows transmitting 1 signal pair up to 4-6MB/s (2 pairs - up to 8-12MB/s). The use of compression will further increase these numbers.

UFS memory chips is have different with eMMC hardware connection signals:

  • RST_n (not use at LS modes), at some devices need connect to VCCQ2
  • REF_CLK (not use at LS modes)
  • DIN_t
  • DIN_c
  • DOUT_t
  • DOUT_c
  • GND
  • VCC(2,8v) , VCCQ (1,8v), VCCQ2 (1,2v)  (Not use wit powered PCB by USB charger)

UFS chips ISP pins marking:

  • TXD0P - DIN0T
  • TXD0N - DIN0C
  • RXD0P - DOUT0T
  • RXD0N - DOUT0C

UFS memory speed standards:

LS modes:

  • PWM-G0: 0.01~3 Mbps
  • PWM-G1: 3~9 Mbps
  • PWM-G2: 6 ~ 18 Mbps
  • PWM-G3: 12 ~ 36 Mbps
  • PWM-G4: 24 ~ 72 Mbps
  • PWM-G5: 48 ~ 144 Mbps
  • PWM-G6: 96 ~ 288 Mbps
  • PWM-G7: 192 ~ 576 Mbps


HS modes:

  • HS-G1: 1.25-1.45Gbps
  • HS-G2: 2.5-2.9Gbps
  • HS-G3: 5.0-5.8Gbp
     

UFS ISP (In-System Programing) connection is released via BGA254 2in1 adapter.

Connect all pins for non powered PCB, or everyone except VCC/VCCQ/VCCQ2 for PCB with power from USB.
