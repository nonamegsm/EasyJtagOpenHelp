Repair eMMC Firmware

Repair eMMC firmware allow for old Samsung eMMC up to eMMC 5.0 revision. Different eMMC have different internal eMMC controller and different firmware structure, ready to upload. The ability to create firmware for new chips is very difficult to implement. Repair eMMC firmware procedure also wiped all eMMC data including RPMB area.

 

Samsung's policy has changed towards creating eMMC firmware without erasing data and digitally signed.
Thus, it is not possible to create firmware for restoring EMMC 5.1 as well as for the latest eMMC 5.0 controller.
Official "ffu" firmware uploading only allows you to update the firmware, but does not help to restore the chip if it's in at "OCR Timeout State".

 

To clean all eMMC data with RPMP area simply use "Update eMMC firmware" from Normal eMMC state.

 

To restore eMMC firmare from "OCR Timeout Error" state need use those steps:

  1. Find TP for eMMC from TP database (in EasyJTAG classic software) or at EasyJTAG Classic software directory and "eMMC TP Pinouts" subfolder
  2. Solder thin wire for TP contact pad
  3. Connect wire to GND
  4. Put eMMC to socket
  5. Check eMMC at software
  6. eMMC will be answer and post info about itself. Revision (Rev.) of eMMC must be 0x56
  7. Press "Update Firmware" button at software and fill full eMMC marking label name
  8. Disconnect TP wire from GND
  9. Press "Confirm" button at software
  10. Restore eMMC firmware will be run

 

List of EasyJTAG firmware restore supported eMMC can see at EasyJTAG Classic software directory and "eMMC Firmware" subfolder.

 

 

NOTE: eMMC firmware restore does not restore the resource of the chip cells, which have a limited number of write-erase cycles.
Thus, if the cells of the chip have exhausted their resource, or are close to the limit, then the life of such a chip is limited to weeks or months.
