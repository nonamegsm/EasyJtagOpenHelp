USB Page

ADB - allows you to work with the device's memory through the ADB interface: Android Debugging Bridge, which is enabled in the device in the Developer Setting > USB Debuging.

 

 

Detect ADB - find/connect to the ADB interface of the device;
Detect Part. - identify device sections;
Dump Part. - read and save the selected section;
Read extCSD - read the configuration of the registers extCSD of the microchips memory device (eMMC) to the file;
Limit Dump Size To - limits the size of the selected section;
Partition List - illustrates the Symlink, Name, Size and Offset of partitions.

 

 

QC EDL tab is working with Qualcomm processors.

 

 

Qualcomm processors have a service mode that can be accessed by shortening a test pins.


Than the processor goes into boot mode and allows user to record the bootloader and trough it to perform the settings.


The sequence of operations :

  1. Port - Allows to select a specific device (phone) that is connected via USB. The button on the right allows to update the list of connected devices and automatically select the first QDLoader 9008 (the phone is in service mode).
  2. Identify - allows to get the device status, ID and hash.
  3. Loader - allows to select one of the compatible (by ID and hash) loaders from EasyJtag database, or use the Custom button to select the user bootloader in the box below.
  4. The next time click the Identify button, a corresponding bootloader will be loaded into the device, allowing user to work with memory.

Reset Device - in some loaders and phones it allows you to switch from service mode to working mode. It doesn't always work, it's the features of a particular phone and Loader;
Read Info - reloading partitions;
Write Target - write to device.   The nuance of WriteMode:
Write Binary Raw to 0 sector - offset linear recording;
Write XML Write by vendor - parses the main firmware file and writes the corresponding files to the specified offsets;
Write by partition name - reads GPT from a device, locates a file with the same name in the selected folder and writes to it;
GPT Backup Writing - backup the GPT;
Read RAW - linear reading;
Read XML - reads files by analogy as they are placed in the firmware;
Unlock User - attempt to reset the user's password;
Stop Operation – stops the device operation.

LG LAF - LG Advanced Flash. LG has developed e technology that greatly simplifies the replacement of system firmware in the device. This technology is called "LAF" (LG Advanced Flash).

 

Initially, LAF technology was developed for LG service centers. In particular, it allowed to restore working capacity of LG mobile devices which do not turn on and do not react to attempts to turn them on (often mobile devices go into this state when trying to increase user's privileges in their operating system or in the case of errors in updating the system software produced by Unqualified users). However, a large number of utilities that use this technology appeared quite quickly outside specialized service centers for flashing the modified system software of LG mobile devices.

 

 

Detect LAF - Detect LG Device Info (Partitions / Android version);
Read RAW - Read dump from LG Device. If you are reading a full dump, you need select "Write binary at RAW mode". Also you need to select a size of write.
Format User - User Custom Data formating;
Clear FRP - FRP Data erasing;
Clear User Lock - User Password / Key erasing;
Stop Operation - button for interrupting / canceling the performed read / clear / write operations.

 

There is an alternative instruction for transferring a LG mobile device to the Download mode published on the site "LG Download Mode utility and documentation".

  • Turn off the mobile device. 
  • Connect the mobile device yo the computer using the USB cable:

 

 

  • Press and hold the Volume Up key.
  • Click on the "Start" ("Power") button.
  • Wait for the mobile device to display the message "Download mode".
  • Release the Volume Up key.
  • You should see the inscription on the screen of the mobile device: "Firmware Update".

 

The image on the LG phone screen switched into the “Firmware Update” mode.

 

In mobile devices with the Android operating system version 6, it is possible to save application data not in the phone’s memory, but in the cloud  https://developer.android.com/guide/topics/data/autobackup.html

 

Therefore, the process of extracting user data from a mobile device may fail due to the fact that initially this data was not stored in the device. In this situation, you can use the functionality of the program “Mobile forensics”, which allows you to restore users’ names and passwords from cloud storage (or to find cloud storage tokens in your device) and access the device data stored in the clouds.
